# NORE 가격 전략 (Pro/Tier 설계)

작성일: 2026-05-17
대상: 창립 파트너 (현재) → Basic/Pro/Business (향후) 마이그레이션

## 1. 핵심 원칙

AI SaaS 표준 패턴(Anthropic, OpenAI, Notion AI, Linear) 차용:
- **사용량 기반 제한 + 명확한 업그레이드 트리거**가 핵심
- 가격은 단순 "기능 ON/OFF"가 아니라 **활성 회원 수 × AI 토큰 사용량 × 부가 기능**의 조합
- 사용자가 자연스럽게 한계에 부딪히면 → Pro 업그레이드 안내 (artificial cap이 아니라 진짜 가치 인지 시점)

## 2. 제한할 핵심 지표 (Metering)

| 지표 | 측정 단위 | 왜 중요한가 |
|------|-----------|-------------|
| **활성 회원 수** | 직전 30일 내 1회 이상 AI 호출/등록 발생한 회원 수 | 트레이너 1명의 실제 운영 부하와 직결. 가장 직관적 |
| **AI 토큰 사용량/월** | 입력+출력 토큰 합산 (월 리셋) | 직접 비용 — 우리 마진 보호 |
| **저장 데이터** | 회원 카톡·식단 사진·KPI 기록 누적량 | 장기적 인프라 비용 |
| **자동화 실행 수/월** | 블로그 자동 발행·SNS 게시·예약 알림 등 트리거 횟수 | 외부 API·인건비 누적 |

### 초과 사용 처리 (Overage)

- **활성 회원 초과**: 31번째 회원부터 자동 추가 안내 (월 ₩3,000/명) → 3명 이상 초과 지속 시 Pro 업그레이드 권유
- **토큰 초과**: 50만 토큰 도달 시 알림 → 100만 토큰 단위로 ₩30,000 추가 구매 OR Pro 업그레이드
- **자동화 초과**: 월 한도 도달 시 다음 달 1일까지 대기 (강제 차단 X, "Pro로 즉시 해제" 안내)

## 5. 마이그레이션 전략

### 5.1 창립 파트너 보호
- 현재 가입한 469명(노어 X EAL 등 포함)의 가격은 **영구 잠금**
- Pro/Business 출시 후에도 Basic 가격(358,800·478,800) 유지
- 단, **사용 한도는 동일 적용** — 50만 토큰 초과 시 업그레이드 권유

### 5.2 신규 가입 (Pro 출시 이후)
- Basic 가격은 동일 유지
- 한도 도달 시 명확한 Pro 업그레이드 경로 제공
- 14일 무료 Pro 체험 (가입 직후 자동 적용, 만료 시 Basic 자동 다운그레이드)

### 5.3 Pro 출시 타임라인 (제안)
- **Phase 1 (현재 ~ 6월 30일)**: Basic만 판매. 사용량 측정 인프라 구축. 한도는 "soft limit"(알림만, 차단 X)
- **Phase 2 (7월)**: Pro 출시. Basic 사용자에게 30일 무료 Pro 체험 자동 부여 → conversion 데이터 수집
- **Phase 3 (10월)**: Business 출시. 다중 지점·협회·프랜차이즈 대상 영업
- **Phase 4 (2027 Q1)**: Soft limit → Hard limit 전환. Overage 자동 과금 시작

## 6. UX 설계 (사이트·앱)

### 6.1 사용량 대시보드 (필수)
- 운영자/트레이너 워크스페이스 좌측 상단에 항상 노출:
  ```
  ┌───────────────────────┐
  │ 토큰 사용량 — 42만 / 50만 │
  │ ███████████░░░ 84%      │
  │ 활성 회원 — 28 / 30      │
  │ Pro로 업그레이드 →       │
  └───────────────────────┘
  ```
- 80% 도달 시 노란색, 95% 도달 시 빨간색 + 인앱 알림

### 6.2 업그레이드 트리거 (자연스러운 시점)
- 토큰 80% 도달 시: "이번 달 활용도가 평소보다 높아요. Pro로 4배 더 쓰세요"
- 회원 25/30 도달 시: "회원 추가가 5명 남았어요. Pro로 100명까지"
- 자동화 80% 도달: "자동화로 시간 절약 중! Pro로 5배 더 돌리세요"

### 6.3 결제 페이지 분기
- 현재 `/checkout/?role=X` → 일반 Basic 결제 (현재 구조 유지)
- 신규 `/checkout/?role=X&plan=pro` → Pro 결제 (Phase 2에서 추가)
- 추천인 코드(EALPRO/EAL5) → Basic + 할인 (현재 유지)
